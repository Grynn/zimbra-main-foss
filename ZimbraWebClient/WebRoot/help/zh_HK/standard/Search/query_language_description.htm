此節詳細說明 @CompanyName@ 的搜尋功能中使用的搜尋語法。有些類型的查詢條件須用來生成介面中常見的視窗。

xxxx搜尋語言結構xxxx

xxxx只需在搜尋欄位中輸入一個單字即可搜尋。xxxx裸詞（不含搜尋運算子的單字）被解釋為在『content』： 運算子中搜尋 -- 與郵件中的任何文字相符。

xxxx更進階的搜尋可透過指定一個xxxxxxxx搜尋運算子xxxxxxxx實現。xxxxxxxx特殊關鍵字後跟一個冒號即為搜尋運算子，其後是適合於該運算子的其他參數。xxxxxxxx例如：xxxx

  • in: inbox，運算子為「in」，而參數為「 inbox」，意即回送名為「 inbox」的資料夾中的郵件。

  • from:someone 運算子是「from」，而參數是「someone」，則會回送寄件人地址中含「someone」的郵件。

在任何關鍵字前都可加上字首「not」，從而指定不符合該條件的項目，例如 not in:inbox。搜尋不區分大小寫，意即「in:inbox」與「in:Inbox」相同。xxxx減號（xxxxxxxx-xxxxxxxx）是 xxxxxxxxNOTxxxxxxxx 的同意詞，因此：xxxxnot in:inbox 與 -in:inbox 相同

xxxx大多數情況下無需在搜尋字串中使用標點符號，因為會被搜尋程式代碼所忽略。xxxxxxxx有些情況下不是這樣（例如，搜尋含有時間 '9:30' 的郵件），在這種情況下你需要為搜尋參數加上引號。xxxxxxxx例如：xxxxsubject:"9:30" 回送的是主題中含字串 9:30 的郵件。

xxxx搜尋參數中允許使用的字元：xxxx

  • xxxx下列字元如果不加引號就不能用於搜尋參數：xxxx ~ ' ! # $ % ^ & * ( ) _ ? / { }[ ] ; :

  • xxxx下列字元可以用於搜尋參數，但不能作為首字使用。xxxx - + < >

xxxx多重搜尋字xxxx

如果輸入了多個搜尋字（用空格分開），預設情況下它們是「 AND」（與）的關係。in:inbox tim 表示「將返回位於收件箱中且 (AND) 含有 tim」的郵件。xxxx對於有多種條件的搜尋項，你可以尋找與其中任意條件相符的項目，也可尋找符合全部條件的項目。xxxxxxxx你可用「進階」搜尋功能進行這兩種類型的搜尋。xxxx

xxxx除「基本」搜尋功能外，所有搜尋窗格的規則為：xxxx

  • xxxx與指定條件中任意條件相符的搜尋稱為 xxxxxxxx'OR'xxxxxxxx（或）搜尋，意思是如果郵件中包含 X 或 Y，則認為該郵件是相符的。xxxxxxxx對於有多個核取方格的「進階」搜尋窗格，可在單一窗格中對多個選項進行選擇，為所需項目建立一個 xxxxxxxx'OR 'xxxxxxxx（或）搜尋。xxxx

  • xxxx同時包含 X 與 Y 的搜尋稱為 xxxxxxxx'AND'xxxxxxxx（與）搜尋，郵件必須符合所有指定的條件才是相符的。xxxxxxxx對於有多個核取方格的「進階」伺服器窗格，在同一窗格中開啟多個例項，並對每個例項中的選項進行不同的剔選，將會使條件被指定為 xxxxxxxx'AND'xxxxxxxx搜尋項。xxxx

只有「OR」出現在查詢條件中。xxxx在「進階」搜尋中進行選擇時，如果作為選項你選擇了在「搜尋」工具列顯示搜尋結果，則「搜尋」框隨即更新並顯示得到的結果。xxxxxxxx對於xxxxxxxx'AND'xxxxxxxx 類型的搜尋，xxxxxxxx'AND'xxxxxxxx一詞並不會出現。xxxx

xxxx提示xxxxxxxx：xxxxAND 和 OR 可結合括號使用。括號中的字詞視為一個單位。例如 from:(john thomas) 等同於 from:john AND from:thomas。如果括號中有 OR，如 from:(john or smith)，則等同於 from:john OR from:thomas。

xxxx在搜尋中使用 xxxxxxxx*xxxxxxxx 作為通配字元xxxx

xxxx星號（xxxxxxxx*xxxxxxxx）可在搜尋中作為通配字元使用，以便找到與單字拼寫相似的內容。xxxx

xxxx請在字首後使用星號 xxxxxxxx*xxxxxxxx。xxxxxxxx例如，搜尋字串 xxxxxxxxdo*xxxxxxxx 返回的結果是 xxxxxxxxdoxxxxxxxx、xxxxxxxxdogxxxxxxxx、xxxxxxxxdoorxxxxxxxx 等。xxxx

xxxx關鍵字說明及例項xxxx

xxxxcontent:xxxx

xxxx用於指定郵件中必須包含的文字。xxxx例如，content:bananas 將找出所有包含「bananas」的條目。

xxxxfrom:xxxx

xxxx用於指定「寄件人」標題中的寄件人姓名或電郵地址。xxxx這可以是文字，如「John Smith III」，可以是電郵地址，如「joe@acme.com」，也可以是網域名稱，如「 zimbra.com」。

xxxxto:xxxx

xxxx於 from: 相同，xxxx只是所指定之人的電郵地址位於「收件人：」標題中指定的名字。

xxxxcc:xxxx

xxxx於 from: 相同，xxxx只是所指定的是郵件「副本：」標題中的收件人。

xxxxsubject:xxxx

xxxx指定的文字必須是在郵件標題中的文字。xxxx例如 subject:new vacation policy。

xxxxin:xxxx

xxxx用於指定一個資料夾。xxxxxxxx例如，xxxxxxxxin:sentxxxxxxxx 應該顯示xxxxxxxx寄件箱xxxxxxxx中的所有條目。xxxx

xxxxhas:xxxx

xxxx用於指定郵件必須具備的屬性。xxxx可指定目標類型包括「attachment」、「phone」或「 url」。xxxx例如，xxxxxxxxhas:attachmentxxxxxxxx將找出所有包含一個或多個附件的郵件。xxxx

xxxxfilename:xxxx

xxxx用於指定附件檔案的名稱。xxxx例如，filename:query.txt 將找出附件檔案名稱為「 query.txt」的郵件。

xxxxtype:xxxx

xxxx用於指定在某個類型的附件中進行的搜尋。xxxx可指定的附件類型包括「text」、「word」、「excel」及「 pdf」。例如，type:word "hello" 將找出含 Microsoft Word 文件附件的郵件，同時在這些文件中搜尋單字「hello」。

xxxxattachment:xxxx

xxxx用於指定包含特定類型附件的任何條目。xxxxxxxx例如，xxxxxxxxattachment:wordxxxxxxxx 將找出所有帶 Word 附件的郵件。xxxx

xxxxis:xxxx

xxxx用於搜尋某種特定狀態下的郵件。xxxx可用的值包括「unread」、「read」、「flagged」、「 unflagged」、「sent」、「draft」、「received」、「replied」、「 unreplied」、「forwarded」、「unforwarded」、「anywhere」、「remote」（在分享資料夾中）、「local」、「sent」。xxxx例如，xxxxxxxxis:xxxxxxxxunread xxxxxxxx將尋找所有未讀的郵件。xxxx

xxxxdate:xxxx

可用此關鍵字指定一個日期，其格式使用瀏覽器本地語言的預設格式（對於美式英文，其格式為 mm/ dd/ yyyy）。例如，date:2/1/2010 將尋找 2010 年 2 月 1 日的郵件。大於 (>) 或小於 (<) 符號可以用於替代 after 或 before。>= 和 <= 也可使用。

xxxxafter:xxxx

xxxx指定某特定日期之後傳送的郵件。xxxx例如，after:2/1/2010 是指 2010 年 2 月 1日之後傳送的郵件。

xxxxbefore:xxxx

xxxx與 xxxxxxxxafter: xxxxxxxx相同xxxxxxxx只是用於指定某特定日期之前傳送的郵件。xxxx

xxxxsize:xxxx

指定郵件的總大小（含附件），以位元組、KB 或 MB 為單位。例如，size:12 kb 將找出大小正好為 12K 的郵件。大於 (>) 或小於 (<) 符號可以用於替代 bigger 或 smaller。

xxxxlarger:xxxx

xxxx與 size: 相似：xxxxxxxx只是指定大於某特定大小的郵件。xxxx

xxxxsmaller:xxxx

xxxx與 size: 相似：xxxxxxxx只是指定小於某特定大小的郵件。xxxx

solo:

找出尚未有回覆的郵件。

xxxxtag:xxxx

xxxx用於尋找被標上指定標記的郵件。xxxx例如，tag:amber 將找出具有「amber」標記的郵件。

priority:

按「高」「低」優先權找出郵件、事項或記事簿內容。例如，priority:high 將找出優先權設定為高的內容。
