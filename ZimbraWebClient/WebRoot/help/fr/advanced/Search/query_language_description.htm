Description du langage de demande

Cette rubrique avancée décrit en détail la syntaxe de recherche utilisée pour la fonction de recherche de Zimbra. Certains types de demandes s’appliquent toujours pour produire l’affichage que vous voyez dans l’interface de Zimbra.

Conseil : Vous pouvez configurer vos Préférences, dans l’onglet Général sur 'Toujours montrer la chaîne de recherche'  pour afficher la recherche en cours dans la barre d’outils de Recherche. Par exemple, si cette option est activée, lorsque vous cliquez sur le dossier Boîte de réception il s’affiche alors la recherche 'dans :boîte de réception'.

Structure de langage de recherche

Les recherches simples peuvent être effectuées en saisissant simplement un mot dans le champ de recherche. Les mots simples (mots sans opérateur de recherche) sont interprétés pour effectuer une recherche dans le « contenu ». Opérateur – ceci correspond à toute zone de texte du mail.

Les recherches plus avancées peuvent être effectuées en spécifiant un opérateur de recherche. Un opérateur de recherche est un mot clé spécial suivi de deux points, puis de tout autre paramètre spécifique à cet opérateur. Par exemple :

  • in:inbox l’opérateur est "in" et le paramètre est "inbox" - ceci permet d’obtenir une correspondance avec les mails qui sont dans le dossier nommé "boîte de réception"

  • from:someone l’opérateur est "from" et le paramètre est "someone" - ceci permet d’établir une correspondance avec les mails contenant le mot "quelqu’un" dans leur adresse mail

Vous pouvez faire précéder un mot clé par le mot "not" afin de spécifier les éléments qui n’ont pas ce critère spécifique, par exemple not in:inbox. La recherche n’est pas affectée par les majuscules, ce qui signifie que "in:inbox" est identique à "in:Inbox". Le signe moins (-) est synonyme de PAS Par conséquent : pas dans :boîte de réception est identique à -dans :boîte de réception

Dans la plupart des cas, il n’est pas nécessaire d’inclure des caractères de ponctuation dans votre caractère de recherche, car ces derniers sont ignorés par le code de recherche. Pour certaines recherches, cela ne s'applique pas (par exemple, rechercher l’heure « 9:30 » dans un mail) et dans ce cas vous devez écrire le paramètre de recherche entre guillemets. Par exemple : subject:"9:30" renvoie les mails comportant le caractère 9:30 dans la ligne d’objet.

Caractères autorisés dans les paramètres de recherche :

  • Les caractères suivants ne peuvent pas être introduits au hasard dans les paramètres de recherche à moins d’être écrits entre guillemets : ~ ' ! # $ % ^ & * ( ) _ ? / { }[ ] ; :

  • Les caractères suivants sont autorisés dans une recherche tant qu’ils ne sont pas placés en tête de la recherche : - + < >

Recherche de plusieurs termes

Si plusieurs termes sont saisis (séparés par des espaces), ils ne sont pas "Ajoutés" par défaut. in:inbox tim signifie "rechercher les mails qui sont dans la boîte de réception ET qui contiennent le mot tim". Pour effectuer des recherches à l’aide de plusieurs critères, vous pouvez rechercher des éléments correspondant à l’un des critères spécifiés ou à l’ensemble des critères. Vous pouvez effectuer les deux types de recherches à l’aide de la recherche avancée.

Pour l’ensemble des volets de recherche autres que la recherche basique, les règles sont les suivantes :

  • La recherche de mails correspondant à l’un des critères spécifiés s’appelle une recherche OR. En effet, si l’un des mails contient le terme X ou Y, il est alors considéré comme une correspondance. Pour les panneaux de recherche avancée à plusieurs options, cocher plusieurs sélections dans un seul panneau permet de créer une recherche OR pour ces éléments.

  • La recherche de mails contenant à la fois X et Y s’appelle une recherche 'AND'. En effet, le mail doit correspondre à l’ensemble des critères spécifiés afin d'être considéré comme une correspondance. Pour les panneaux de serveur avancé ayant des cases activées, ouvrir plusieurs instances du même panneau et sélectionner plusieurs options dans chacune d'entre elles permet de spécifier les critères comme une recherche 'AND'.

Seul "OR" s’affiche dans une demande. Si vous avez sélectionné comme option d'afficher la demande de recherche dans la barre de recherche (lorsque vous effectuez des sélections dans la recherche avancée), la zone de recherche s'actualise pour afficher les résultats de la demande. Avec le type de recherche 'AND', le mot 'AND' ne s’affiche pas.

Conseil : En utilisant des parenthèses avec AND et OR, les mots entre parenthèses sont considérés comme une unité. Par exemple from: (john thomas) est équivalent à from: john AND from:thomas. Si vous utilisez OR dans les parenthèses, from:(john or smith), la recherche correspond aux résultats from:john OR from:thomas.

Utiliser le caractère * de substitution dans une recherche

L’astérisque (*) peut servir de caractère de substitution pour rechercher dans une zone de texte des mots ayant une orthographe similaire.

Utiliser l’astérisque * en tant que caractère de substitution après un préfixe. Par exemple, le caractère de recherche ca* renvoie aux éléments tels que cache, caprice, carotte, etc.

Descriptions des mots-clés et exemples

Contenu :

Spécifie le texte que le message doit contenir. Par exemple, content:bananes recherche tous les éléments contenant le mot "bananes".

from:

Spécifie le nom ou l'adresse mail d’un expéditeur qui se trouve dans l’en-tête De. Il peut s’agir de texte, comme pour "John Smith III", d’une adresse mail comme "joe@acme.com", ou d’un domaine comme par exemple "zimbra.com".

to:

Identique à from: hormis le fait qu’il spécifie une des personnes à laquelle le mail était adressé dans l’en-tête To:.

cc:

Identique à from: mis à part qu’il spécifie un destinataire dans l’en-tête Cc: du mail.

subject:

Spécifie une zone de texte qui doit apparaître dans l’en-tête d’objet du mail. Exemple : subject:"nouvelle politique de congés". Utiliser des guillemets pour rechercher une correspondance exacte.

in:

Spécifie un dossier. Par exemple, in:sent vous permet d’afficher tous les éléments contenus dans votre dossier Envoyés.

has:

Spécifie un attribut que le mail doit posséder. Les types d’objets que vous spécifiez sont : "attachment", "téléphone", ou "url". Par exemple, has:attachment vous permet de rechercher tous les mails contenant une ou plusieurs pièces jointes de n'importe quel type.

filename:

Spécifie le nom d’une pièce jointe. Par exemple , filename:query.txt permet de rechercher les mails ayant une pièce jointe nommée "query.txt".

type:

Spécifie une recherche dans les pièces jointes d’un type spécifique. Les types de pièces jointes que vous pouvez spécifier sont : "text", "word", "excel", et "pdf". Par exemple, type: word "salut" permet de rechercher les mails avec des pièces jointes qui sont des documents Microsoft Word. La recherche s'effectue dans ces pièces jointes par rapport au mot "salut".

attachment:

Spécifie tout élément d’un certain type de pièce jointe. Par exemple, attachment:word vous permet de rechercher tous les mails ayant des pièces jointes Word.

is:

Recherche les mails avec un statut spécifique. Les valeurs autorisées sont "non lu", "lu", "avec drapeau", "sans drapeau", "envoyé", "brouillon", "reçu", "répondu", "non répondu", "transféré", non transféré", "partout", "distant" (dans un dossier partagé), "local", "envoyé". Par exemple, is: unread vous permet de rechercher tous les mails non lus.

date:

Utilisez ce mot clé pour spécifier une date, en utilisant le format de date par défaut de votre navigateur (pour le français, le format est jj/mm/aaaa). Par exemple, date:2/1/2008 permet de rechercher les mails datés du 1er février 2008. Les symboles supérieur à (>) ou inférieur à (<) peuvent être utilisés en remplacement de après ou avant. >= et <= sont également autorisés.

after:

Spécifie un mail envoyé après une certaine date. Par exemple, after:2/1/2008 spécifie les mails envoyés après le 1er février 2007.

before:

Similaire à after: hormis le fait qu’il définit les mails envoyés avant la date spécifiée.

size:

Spécifie les mails dont la taille totale, y compris les pièces jointes, est d’un certain nombre d’octets, de kilo-octets, ou de méga-octets.  Par exemple, size:12 ko permet de rechercher les mails dont la taille est exactement de 12 Ko. Les symboles supérieur à (>) ou inférieur à (<) peuvent être utilisés à la place de plus grand ou plus petit.

larger:

Identique à size: hormis le fait qu’il permet de spécifier une taille supérieure à celle indiquée.

smaller:

Identique à size: hormis le fait qu’il permet de spécifier une taille inférieure à celle indiquée.

tag:

Recherche les mails marqués par un tag spécifique. Par exemple, tag :ambre permet de rechercher les mails marqués d’un tag appelé "ambre".
