Установка параметров для окна создания сообщения

На вкладке «Настройки > Создание сообщений» используйте следующие параметры.

  • Написать письмо. Выберите текстовый редактор по умолчанию: для HTML или для обычного текста.

Установите настройки шрифта по умолчанию, используемые с кодом HTML для создания в почте, документах и ежедневнике.  Можно определить стиль, размер и цвет шрифта, используемые по умолчанию.

  • Создаваемые сообщения автоматически сохраняются как черновики через каждые несколько минут. Если не нужно сохранять черновики автоматически, эту функцию можно отключить. Чтобы сохранить черновик сообщения вручную, нажмите кнопку «Сохранить черновик».

  • Ответ/пересылка в формате исходного сообщения. Даже если формат используемого по умолчанию текстового редактора отличается от формата полученного сообщения, то при установке этого флажка ответное сообщение будет в том же формате, что и полученное. Это необходимо для того, чтобы получатели могли получать сообщение в наиболее удобном для них формате.

  • Если установлен параметр Всегда создавать сообщение в новом окне, то для создания сообщения открывается новое окно, отдельное от почтового ящика Zimbra. По умолчанию для создания сообщения открывается вкладка «Создание сообщения» на панели инструментов «Приложение». В обоих случаях при открытом окне создания сообщения можно просматривать почтовый ящик и перемещаться в нем.

  • Обязательно проверять правописание перед отправкой сообщения. Установите этот параметр для автоматической проверки правописания перед отправкой сообщения. При обнаружении ошибок открывается диалоговое окно, в котором указывается количество ошибок и предлагается исправить их перед отправкой.  

  • Отправленные сообщения. По умолчанию включен параметр «Сохранить копию в папке "Отправленные"». Копии отправленных сообщений сохраняются в папке «Отправленные». Если в этом нет необходимости, можно установить флажок «Не сохранять отправленные сообщения».

  • Ответить/Ответить всем. Укажите, следует ли включать текст исходного сообщения в тело ответа. Можно включать или не включать текст исходного сообщения, включать его как вложение, включать его в тело письма с выбранным префиксом, включать только самое последнее сообщение.  (Другими словами, будет процитирован только текст, написанный отправителем сообщения, но не предыдущий текст). Если включен параметр Включить заголовки, то в вашем сообщении будут отображаться данные заголовков «Кому» или «От» и «Тема».

  • Переслать. Укажите, следует ли включать текст исходного сообщения в тело пересылаемого сообщения. Можно выбрать вариант «Включить исходное сообщение и заголовки с префиксом» или «Включить исходное сообщение как вложение». Если включен параметр Включить заголовки, то в вашем сообщении будут отображаться данные заголовков «Кому» или «От» и «Тема».

  • Префикс. Укажите, следует ли ставить префикс «>» или «|» перед каждой цитируемой строкой предыдущих сообщений, которые пересылаются или на которые дается ответ.
