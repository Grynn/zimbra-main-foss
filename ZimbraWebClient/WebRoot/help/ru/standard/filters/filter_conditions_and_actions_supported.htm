Поддерживаемые условия и действия фильтров

Регистр букв правил фильтра не имеет значения, т.е. в правилах не учитываются заглавные буквы.

Условия

Поля сравнения включают следующие данные.

  • От. Используется для указания имени в заголовке сообщения «От:».

  • Кому. Используется аналогично параметру «От:», но здесь указываются имена из заголовка «Кому:».

  • Копия. Используется аналогично параметру «От:», но здесь указываются данные из заголовка «Копия:».

  • Тема. Указываются данные из заголовка «Тема:».

  • Именованный заголовок. При выборе этого параметра появляется дополнительное поле ввода текста перед оператором сравнения (часть «содержит»). Этот параметр позволяет указывать любой заголовок сообщения. Можно указать не только стандартные поля «Кому:» или «От:», но также дату, получателя и другие специальные поля, которые могут быть включены в заголовок сообщения. Этот параметр можно также использовать для фильтрации исходящих сообщений с «испорченными» заголовками, т.е. заголовками, которые не содержат стандартную для электронных сообщений информацию. Иногда в спаме, который генерируется автоматически, отсутствует информация в таких полях, как «Отправитель» или «Получатель». Для указания проверяемого заголовка используется второе текстовое поле (поле, которое находится справа от оператора сравнения).

  • Размер. Этот параметр используется для указания сообщения, размер которого больше или меньше указанного. Размер сообщения включает любые файловые вложения. Этот параметр можно использовать для удаления сообщений слишком большого размера.

  • Дата. Этот параметр используется для указания сообщения, отправленного до или после определенной даты.

  • Тело письма. Параметры для тела письма «содержит» или «не содержит» позволяют указать соответствующие слова в теле письма. Устанавливать фильтры для слов в файловых вложениях нельзя.

  • Вложение. Можно установить фильтр для сообщений, которые содержат или не содержат файловые вложения.

  • Адрес в. Проверка наличия адреса в контактах. В следующем поле можно указать, какой адрес.

Операторы сравнения включают следующие данные.

  • Содержит/не содержит. Этот параметр указывает, что строка темы должна содержать указанную подстроку.  Например, если указать Тема содержит яблоко, то ему может соответствовать «Я хочу яблоко».

  • Точно соответствует/не соответствует шаблону. Подстановочный знак звездочка (*) используется в поле сравнения почтового фильтра для подстановки одного или более знаков фильтра. Например, если указать, что заголовок должен соответствовать слову «яблоко», то фильтроваться будут слова «яблоки», «яблоки-пироги», но не «мои яблоки».

Действия

  • Хранить в папке «Входящие». Сохраняет сообщения в папке «Входящие». Если почтовое сообщение не соответствует ни одному из правил фильтра, это действие выполняется по умолчанию.

  • Записать в папку. Перемещает сообщения в указанную папку.

  • Отклонить. Удаляет сообщение, не доставляя его. Сообщения нет ни в одной папке, даже в Корзине.

  • Переслать. Сообщения перенаправляются на указанный адрес.

  • Пометить. Выберите вариант Прочитанное или С флагом.

  • Пометить тегом. Соответствующие сообщения можно пометить указанным тегом.

Несколько действий

В правиле фильтра может быть определено несколько действий. Сочетание действий толкуется следующим образом.

  • Отклонить. В сочетании с другими действиями этот параметр игнорируется, выполняются другие действия.

  • Записать в папку. В результате нескольких действий «файл в» несколько копий сообщения сохраняются в разных папках. Если указанная папка не существует, сообщение сохраняется в папке «Входящие».

  • Тег/пометить. Эти действия применяются к сообщению, к которому применялось последнее действие. В случае нескольких действий «файл в» какие-то копии сообщений сохраняются с тегами, какие-то — без.

  • Хранить в папке «Входящие». Можно определить несколько действий «сохранить», но при этом только одна копия сообщения будет сохраняться в папке «Входящие».

  • Переслать по адресу. Сообщения будут пересылаться по указанному адресу.
