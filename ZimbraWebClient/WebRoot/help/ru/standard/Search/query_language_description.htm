Описание языка запросов

Этот расширенный раздел подробно описывает грамматику языка поиска функции поиска веб-клиента Zimbra. Для получения вида, который пользователь видит в интерфейсе Zimbra, применяется определенный тип запроса.

Совет. На вкладке «Настройки->Общие» можно установить флажок Всегда показывать строку поиска, чтобы видеть текущий запрос на панели инструментов поиска. Например, если включен этот параметр, щелчок папки «Входящие» приведет к отображению строки запроса 'in:inbox'.

Структура языка поиска

Простой поиск можно выполнить, просто введя слово в поле поиска. Только слова (слова без оператора поиска) используются для поиска в операторе «content:», что соответствует любому тексту в сообщении.

Расширенный поиск можно выполнить, указав оператор поиска. Оператор поиска — это специальное ключевое слово, за которым следует двоеточие, а затем другие параметры, специфические для этого оператора. Например:

  • оператором в in:inbox является «in», а параметром — «входящие», что выдает сообщения, которые содержатся в папке по имени «входящие».

  • оператором в from:petrov является «от», а параметром «petrov», что выдает сообщение, которые содержат слово «petrov» в электронном адресе.

Перед любым ключевым словом можно поставить префикс «не», чтобы определить элементы, которые не содержат критерий, например not in:inbox. В профиле поиска не учитывается регистр букв, это значит, что «in:inbox» равнозначно «in:Inbox». Знак минуса (-) является синонимом NOT Поэтому: not in:inbox равнозначно -in:inbox.

В большинстве случаев нет необходимости включать знаки пунктуации, строку поиска, поскольку они игнорируются кодом поиска. Но в определенных случаях это не так (например, поиск времени «9:30» в сообщении), и в этих случаях параметр поиска следует заключать в кавычки. Например: subject:«9:30» выдает сообщения, которые содержат в теме «9:30».

Допустимые знаки в параметре поиска.

  • Следующие знаки можно использовать в параметре поиска только заключенными в кавычки: ~ ' ! # $ % ^ & * ( ) _ ? / { }[ ] ; :

  • Следующие знаки можно использовать в параметре поиска только, если они не являются первыми знаками: - + < >

Несколько условий поиска

Если вводится несколько условий поиска (разделенных пробелами), по умолчанию они объединяются оператором «AND». in:inbox tim означает «выдать сообщения, которые находятся в папке «Входящие» И содержат слово «tim». Для поиска с использованием нескольких критериев, можно найти элементы соответствующие одному или всем указанным критериям. Можно выполнить оба типа поиска с использованием функции создания расширенного поиска.

Для всех областей поиска, кроме обычного поиска, правила следующие.

  • Поиск сообщений, соответствующих любым указанным критериям называется поиском с условием 'OR', потому что, если сообщение содержит X или Y, оно считается соответствующим условиям поиска. Для панелей расширенного поиска с несколькими установленными флажками выбор нескольких вариантов на одной панели создает поиск для этих элементов с условием OR.

  • Поиск сообщений, которые содержат X и Y называется поиском с условием AND, потому что сообщение должно соответствовать указанным критериям, чтобы считаться соответствием. Для панелей расширенного сервера с установленными флажками, открытие нескольких копий одной и той же панели и установка разных флажков на каждой из них делает необходимым определение критериев поиска как поиска с условием AND.

В запросе появится только «OR». Если задан параметр показывать запрос поиска на панели поиска по мере выбора в поле «Расширенный поиск», текстовое окно поиска будет обновляться и отображать полученный запрос. В профиле поиска типа AND слово AND не отображается.

Совет. Использование скобок с операторами AND и OR. Слова в скобках считаются единицей. Например, from: (иван сидоров) эквивалентно from:иван AND from:сидоров. Если использовать OR в скобках, from:(иван or иванов), результат поиска будет from:иван OR from:сидоров.

Использование * в качестве подстановочного знака в поиске

Звездочку (*) можно использовать в качестве подстановочного знака при поиске содержимого, которое содержит слова с похожим написанием.

Используйте звездочку (*) как подстановочный знак после префикса. Например, строка поиска ты* выдает элементы ты, тыква, тысяча и т.д.

Примеры и описания ключевых слов

content:

Определяет текст, который должен содержаться в сообщении. Например, content:яблоки находит все элементы, которые содержат слово «яблоки».

from:

Определяет имя или электронный адрес отправителя, которые содержатся в заголовке «От:». Это может быть текст «Иван Иванов III», электронный адрес «ivan@kontora.ru» или домен «zimbra.com».

to:

Аналогично условию «От:» за исключением того, что здесь определяются получатели, указанные в заголовке «Кому:».

cc:

Аналогично условию «От:» за исключением того, что здесь определяется получатель из заголовка сообщения «Копия:».

subject:

Определяет текст, отображаемый в заголовке сообщения «Тема». Примером может быть subject:"новая политика отпусков". Для поиска точного соответствия используются кавычки.

in:

Определяет папку. Например, in:отправленные выдаст все элементы в папке Отправленные.

has:

Определяет атрибут, который должен содержаться в сообщении. Типы объектов, которые можно определить: «attachment», «phone» или «url». Например, если указать has:attachment будут найдены все сообщения, которые содержат одно или несколько вложений любого типа.

filename:

Определяет имя файла вложения. Например, если указать filename:query.txt будут найдены все сообщения с вложением с именем «query.txt».

type:

Определяет поиск во вложениях определенного типа. Типы вложений, которые можно определить: «текст», «word», «excel» и «pdf». Например, если указать type:word «hello», будут найдены сообщения с вложениями формата Microsoft Word, содержащие слово «hello».

attachment:

Определяет элемент с определенным типом вложения. Например, если указать attachment:word, будут найдены все сообщения с вложениями формата Word.

is:

Поиск сообщений с определенным статусом. Допустимые значения: «unread», «read», «flagged», «unflagged», «sent», «draft», «received», «replied», «unreplied», «forwarded», «unforwarded», «anywhere», «remote» (в общей папке), «local». Например, если указать is: unread, будут найдены все непрочитанные сообщения.

date:

Это ключевое слово используется для определения даты с использованием формата принятого по умолчанию в используемом браузере (для русского языка формат следующий: dd.mm.yyyy). Например, если указать date:1/2/2007, будут найдены сообщения датированные 1 февраля, 2007. Можно использовать знаки больше (>) или меньше (<) вместо после или до. >= и <= также допустимы.

after:

Определяет сообщения отправленные после определенной даты. Например, after:1/2/2007 определяет сообщения, отправленные после 1 февраля, 2007.

before:

Аналогично условию after:, но определяет сообщения, отправленные до определенной даты.

size:

Определяет сообщения, общий размер которых, включая вложения, соответствует определенному объему в байтах, килобайтах или мегабайтах. Например, если указать size:12 kb будут найдены сообщения с точным размером 12 КБ. Вместо слов «больше» или «меньше» можно использовать знаки больше (>) или меньше (<).

larger:

Аналогично условию размера, но определяет размер больше указанного.

smaller:

Аналогично условию размера, но определяет размер меньше указанного.

tag:

Находит сообщения, помеченные определенным тэгом. Например, если указать tag:amber, будут найдены сообщения с тегом, который называется «Амбра».
