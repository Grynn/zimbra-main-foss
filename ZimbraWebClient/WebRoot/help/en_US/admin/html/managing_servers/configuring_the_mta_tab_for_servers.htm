Configuring the MTA Tab for Servers

The following MTA settings for servers can be changed. The server settings override the global settings.

 

Enable authentication

Enables SMTP client authentication, so users can authenticate. Only authenticated users or users from trusted networks are allowed to relay mail.

 

TLS authentication only

When checked, forces all SMTP auth to use Transaction Level Security (similar to SSL) to avoid passing passwords in the clear.

 

Web mail MTA Hostname

The MTA that the web server connects to for sending mail.

 

Web mail MTA port

The default is port 25. This cannot be changed from the administration console.

 

Web mail MTA timeout (s)

This setting determines how long the MTA will try to connect to the server.

 

Relay MTA for external delivery

Enter the relay host name. This is the Zimbra MTA to Gateway host.

 

Enable DNS lookups

The Zimbra MTA makes an explicit DNS query for the MX record of the recipient domain. The default is enabled.  If this option is disabled, set a relay host in the Relay MTA for external delivery field.
