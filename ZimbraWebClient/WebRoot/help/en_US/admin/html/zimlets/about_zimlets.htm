Managing Zimlets

Zimlets™ is a mechanism for integrating the Zimbra Collaboration Suite with third party information systems and content. Zimlets are also used to intermix user interface within the Zimbra Web Client for Email, Calendar, Address Book, and Documents applications. You can create other Zimlets so that users can interact with your company resources or other defined applications from the Zimbra Web Client.

    • For information about how to develop and configure Zimlets see the Zimlet Development on the Zimbra wiki.  Go to the Zimbra gallery, http:// gallery.zimbra.com, to see the type of Zimlets that can be used within the ZCS.  

    From the administration console, you can add new Zimlets, set access privileges by COS and by individual accounts, and disable and uninstall Zimlets from ZCS.  Users can disable and enable Zimlets that are not mandatory from their Preferences>Zimlets page.

    If you create a Zimlet that needs additional configuration after deployment, you will need to use the ZCS CLI to access and change the Zimlet configuration. See the Working with Zimlets chapter in the Administrator's Guide for details.

    Zimlets that are deployed are listed on the Zimlets contents pane. You can remove Zimlets, disable and enable them from the Zimlets content pane.

    Default Zimlets Enabled on ZCS

    ZCS includes pre-configured Zimlets that enhance the user experience while working in the Zimbra Web Client. These Zimlets are already deployed and made available from the COS.  You can manage which Zimlets are available to users from the COS Zimlets tab.

    • com_zimbra_date. When users click on a date either in the email or on the mini-calendar, their calendar schedule for that date displays.

      • If you disable the date Zimlet, the QuickAdd feature and the ability to click on a day and go to that date in the calendar does not work.

      • com_zimbra_email. Highlights and previews associated contact for an email address. Users can see complete contact information if it is available in their address books.

      • com_zimbra_url. This highlights and links URLs. Users can quickly go to the web site specified in an email message.

       

      • com_zimbra_phone. Users can click on a phone number that displays in any of the application pages to quickly call that number if they have the installed a VOIP software application such as Skype or Cisco VOIP.  When they click on the phone number, the VOIP application is launched.
