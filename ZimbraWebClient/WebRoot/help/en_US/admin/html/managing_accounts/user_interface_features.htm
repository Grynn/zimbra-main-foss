Features that add functionality to accounts

The Zimbra Web Client interface features that can be enabled by either COS or Account configuration are listed below. In addition to these features, additional Preferences and Advanced features can be enabled to enhance how some of these features work.

Zimbra Web Client User Features

Feature

Description

Mail

Users can receive, create and manage email messages. See ZCS Communication Features.

Address Book

Users can create their own contacts lists. When this is enabled, a Contacts and an Emailed Contacts address book are created in the user's Address Book. See ZCS Communication Features.

Calendar

Users can use the Calendar and scheduling tool to let users maintain their calendar. See ZCS Communication Features.

Documents

Users can create and organize web documents. See ZCS Communication Features.

Tasks

Users can create and organize tasks. See ZCS Communication Features.

Briefcase

Users can upload files and access the files from ZWC. See ZCS Communication Features.

Tagging

Users can create tag labels and apply the tag to messages. Users can apply multiple tags to a single item to classify it under several categories simultaneously.   

Sharing

Users can create tag labels and apply the tag to messages. Users can apply multiple tags to a single item to classify it under several categories simultaneously.   

Change Password

This requires users to change their log in password the first time they log in after the administrator changes the password

Change UI Themes

Users can change the background colors in their view of Zimbra Web Client.  The themes that are available are  configured in the COS Themes tab.

Advanced search

Users can build detailed search request that include looking up mail by date, domain, flag, object, size, attachment, and time.     

Saved searches

Users can save a search criteria that they built with Advanced search to quickly run the search again.  

Initial search preference

Users can select which of their folders to display when they log on. Also, this is the folder that is searched before other folders.

IMAP access

Enable users to use third party mail applications, such as Thunderbird or Outlook to access their mailbox using the IMAP protocol.

POP3 access

Post Office Protocol. Enable users to use third party mail applications, such as Thunderbird or Outlook to access their mailbox using the POP3 protocol

External POP3 access

Users can set up to retrieve their POP accounts email messages directly in Zimbra Web Client. When they retrieve their POP email messages, the messages and attachments are saved on the Zimbra server.

Allow users to specify a forwarding address  

Users can specify an address to forward mail they receive. You can check Don't keep a local copy of messages, and a copy of the forwarded email message is not kept in the ZWC mailbox. When this feature is enabled in the COS, when you configure the account, you can specify a default forwarding address.

Conversations

Messages can be displayed grouped into conversation or as a message list.  Conversations group messages by subject. If this feature is turned on, conversation view is the default. Users can change the default from their account Preferences tab.

Mail Filters

Users can define a set of rules and corresponding actions to apply to incoming mail. When an incoming email message matches the conditions of a filter rule, the corresponding actions associated with that rule are applied.  The mail filter quota for an account is set at 21K. The mail filter quota cannot be changed. If users create filters or edit existing filters that cause the mail filter size to reach 21K, an error is displayed and the changes cannot be saved. Existing filters will continue to work. Users can delete filters to maintain their quota.

HTML compose

Permits users to compose rich emails that may contain different fonts, colors, and other style choices.

Sending auto-reply message

Users can create an email message that automatically replies to incoming messages.  This is commonly used as a vacation message. A message is sent to each recipient only once every seven days, regardless of how many messages that person sends to the address during that week. Users can set the start and stop dates for sending their message.

New mail notification

When this is checked, users can specify an email address that receives notification that there is email in the Zimbra mailbox.  In the preferences section, you can enable whether the New mail notification is on or off and set the address. Users can also do this from Options on the Zimbra Web Client.

Persona

Users can create multiple mail identities, called personas, which can be applied to email messages they send. For each identity, they set up an email address and name that appears in the From header of messages they send. They can also set up their accounts to receive and reply to external  POP and IMAP accounts from their ZWC account.

GAL access

Users can access the Global Address List.

Autocomplete from GAL

As users type in the address field, suggested addresses from GAL that match the text are displayed. Users can select an address rather than type the complete address.

Keyboard shortcuts aliases

When this is enabled, users can create their own keyboard shortcuts to quickly navigate in folders, tags, and searches.  If this is not enabled, users do not see these tabs in the Preferences Shortcuts tab.

Enable keyboard shortcuts must be enabled to use Keyboard shortcuts

Flag

Users can flag messages. Flags can be used to indicate an action-needed item or to otherwise distinguish the message or conversation.  

Instant notification

When this is enabled, new mail, new calendar events, etc are pushed to the ZWC in real time.  Instant Notification must be enabled for Instant Messaging.

Group calendar

Group calendar must be enabled to have full Calendar functionality to schedule appointments and share a calendar. When Group calendar is disabled, Calendar becomes a personal calendar only. The user can create their appointments, but cannot schedule meetings or share calendars with others.

Initial search preference

Users can select which of their folders to display when they log on. Also, this is the folder that is searched before other folders.
