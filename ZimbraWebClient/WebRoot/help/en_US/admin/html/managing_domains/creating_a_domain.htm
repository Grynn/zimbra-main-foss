Adding a New Domain

One domain is identified during the installation process and additional domains can be added. When you add a domain you configure the Global Address List (GAL) mode and the Authentication mode for the domain. You can also set up virtual hosts, create the domain Documents account, and limit the number of accounts that can be created on the domain.

To add a domain, click New on the Domain toolbar. The Create New Domain wizard walks you through the steps to add the new domain.

  1. On the General Information dialog, enter the complete domain name, description, the default Class of Service (COS) for the domain, and notes you want to add about the domain. You can also set the maximum number of accounts that can be configured for this domain.  Click Next.

    • If you do not enter a specific COS, the default COS is used.

  2. Choose the GAL mode, either Internal, External, or Both. The default is Internal.

  3. (Optional) Change the maximum number of search results returned for a GAL Search.  The default is 100 results. Click Next.

  4. If you selected Internal as the GAL mode, the next step is to configure the authentication mode.

If you selected External or Both, the next step is to configure the LDAP server

  1. If the authentication mechanism is Internal, no other authentication mode configuration is required.

If the authentication mechanism is External LDAP or External Active Directory, the next step is to configure the authentication mechanism.

  1. Select either External LDAP or External Active Directory.

  2. Enter the external server name

  3. For External Active Directory, enter the active directory domain name. The external server port is 389.

  4. If you will use SSL, check the box to Use SSL to connect to the server.

  5. For external LDAP, enter the LDAP Bind Template name.

  6. The Authentication Settings summary page is displayed with the information you just entered. Enter a username and password to test the authentication settings. Click Test.

  7. The Authentication Test Results page displays the test results. If the results failed, click Previous, to resolve the issues described in the Details text box. Click Finish, if the test past.

 

 

Click Next.

  1. (Optional) Set up the virtual hosts. Enter the virtual host addresses. Click Next.

  2. (Optional) Create the domain level Documents wiki account.

    1. Check the box to create domain Documents.

    2. Enter the Documents account name and password. The default account name is "wiki", but you can change this.  Click Next.

    3. On the Documents Access Control dialog, define the global access control privileges.

      • By default all users in the domain can read, write, delete, and insert (add) new pages in the Domain Documents wiki.

      • Users in all domains can be given read, write, delete, and insert permissions.

      • Public can be given read-only access to the Documents pages.

      • Members of distribution lists can be given read, write, delete, and insert permissions.

  3. When Finish is clicked, the domain is created.
