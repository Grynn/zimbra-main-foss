Adding a New Domain

One domain is identified during the installation process and additional domains can be added. When you add a domain you configure the Global Address List (GAL) mode and the Authentication mode for the domain. You can also set up virtual hosts, create the domain Documents account, and limit the number of accounts that can be created on the domain.

To add a domain, click New on the Domain toolbar. The Create New Domain wizard walks you through the steps to add the new domain.

  1. On the General Information dialog, enter the following domain information.

    • Domain name. Enter the full name of the domain (domainexample.com)

    • Public service host name. Enter the host name of the REST URL. This is the URL that is used when sharing Documents Notebooks, email folders and Briefcase folders as well as sharing Task lists, Address Books, and Calendars. See Setting up a Public Service Host Name in the Managing ZCS Configurations chapter in the ZCS Administrator's Guide.

    • Inbound SMTP host name. If your MX records point to a spam-relay or any other external non-Zimbra server, enter the name of the server here.

    • Description. Text field that displays in the Domain list in the Content pane.

    • Maximum accounts for this domain. Enter the maximum number of accounts that can be created on this domain.  The number entered here is displayed in the Account Limits tab.  The default is no limit (blank field)

    • Default Class of Service (COS) for the domain. If you do not enter a COS, the default COS is used. The COS entered here is  automatically assigned to accounts created on the domain if no other COS is entered.

    • Domain status. The domain status is active in the normal state. You may want to change this status when you setting up the domain if you are not ready to use the domain.

    • Notes you want to add about the domain. You can also set the maximum number of accounts that can be configured for this domain.  

Click Next.

  1. Global Address List Mode. Choose the GAL mode, either Internal, External, or Both. The default is Internal. (Optional) Change the maximum number of search results returned for a GAL Search.  The default is 100 results. Click Next.

  2. Authentication Mode. If you selected Internal as the GAL mode, the next step is to configure the authentication mode.

    • If you selected External or Both, the next step is to configure the LDAP server

    • If the authentication mechanism is Internal, no other authentication mode configuration is required.

  • If the authentication mechanism is External LDAP or External Active Directory, the next step is to configure the authentication mechanism.

    1. Select either External LDAP or External Active Directory.

    2. Enter the external server name

    3. For External Active Directory, enter the active directory domain name. The external server port is 389.

    4. If you will use SSL, check the box to Use SSL to connect to the server.

    5. For external LDAP, enter the LDAP Bind Template name.

    6. The Authentication Settings summary page is displayed with the information you just entered. Enter a username and password to test the authentication settings. Click Test.

    7. The Authentication Test Results page displays the test results. If the results failed, click Previous, to resolve the issues described in the Details text box. Click Finish, if the test past.

     

     

Click Next.

  1. Virtual Hosts. (Optional) Enter the virtual host address. Virtual hosting allows you to host more than one domain name on a server. When you create a virtual host, this becomes the default domain user login. Click Next.

  2. Documents. (Optional) Create the domain level Documents wiki account. Documents is a document sharing and collaboration application. See About Zimbra Documents

    • Check the box to create domain Documents.

    • Enter the Documents account name and password. The default account name is "wiki", but you can change this.  Click Next.

    • On the Documents Access Control dialog, define the global access control privileges.

      • By default all users in the domain can read, write, delete, and insert (add) new pages in the Domain Documents wiki.

      • Users in all domains can be given read, write, delete, and insert permissions.

      • Public can be given read-only access to the Documents pages.

      • Members of distribution lists can be given read, write, delete, and insert permissions.

  3. When Finish is clicked, the domain is created.
