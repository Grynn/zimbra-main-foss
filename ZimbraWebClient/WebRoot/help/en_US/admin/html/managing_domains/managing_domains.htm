Managing Domains

One domain is identified during the installation process and additional domains can be added from the administration console.

When you create a new domain, the Create New Domain wizard guides you through the required steps. After a domain is created you can edit the configuration.

To edit a domain, click Domains and select the domain. The Domain toolbar becomes active. From the toolbar you can configure GAL and Authentication directly, or you can click Edit to access the Domain configuration tabs.

For each domain, you can configure the following:

  • Domain General settings, including domain name, description, the number of accounts that can be created on the domain, and default Class of Service assigned to the domain.

  • Domain GAL settings. Options are Internal LDAP server, External LDAP server or Both for GAL lookups and the number of results returned for a GAL search.

  • Domain Authentication settings. Options are Internal, External LDAP, or External Active Directory

  • Virtual Hosts that are used to establish a default domain for user login.

  • The domain Documents account and who has permissions to access the Documents pages.

To change a domain name use the CLI utility, zmprov.
