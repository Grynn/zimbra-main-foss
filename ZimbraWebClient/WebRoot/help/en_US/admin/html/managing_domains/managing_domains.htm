Managing Domains

One domain is identified during the installation process and additional domains can be added from the administration console.

When you create a new domain, the Create New Domain wizard guides you through the required steps. After a domain is created you can edit the domain's configuration. Click Domains in the Navigation pane and select the domain. The Domain toolbar becomes active. From the toolbar you can configure GAL and Authentication directly, or you can click Edit to access the Domain configuration tabs.

For each domain, the following settings can be configured:

  • General Information. Time zone for the domain , public service host name, inbound SMTP host name, default class of service for the domain and the status of the domain  

    • Public service host name. Enter the host name of the REST URL. This is the URL that is used when sharing Documents Notebooks, email folders and Briefcase folders, as well as sharing Task lists, Address Books, and Calendars. See Setting up a Public Service Host Name in the Managing ZCS Configurations chapter in the ZCS Administrator's Guide.

    • Inbound SMTP host name. If your MX records point to a spam-relay or any other external non-Zimbra server, enter the name of the server here.

    • Description. Text field to write a brief description of the domain. This displays in the Domain list in the Content pane.

    • Domain status. The domain status is active in the normal state. You may want to change this status when you setting up the domain if you are not ready to use the domain.

    • Default  Class of Service (COS) for the domain. If you do not enter a COS, the default COS is used. The COS entered here is  automatically assigned to accounts created on the domain if no other COS is entered.

  • GAL. Shows the GAL setting for the domain. This can be  Internal LDAP server, External LDAP server or Both.  The number of results returned for a GAL search can be configured. If this field is blank, no maximum is configured.

  • Authentication. Shows the configured authentication type for the domain. This can be Internal, External LDAP, or External Active Directory

  • Virtual Hosts. Configure virtual hosts that are used to establish a default domain for user login.

  • Briefcase. Configure the company name that is displayed in the prompt on the Authentication Required dialog used to log into Briefcase folders shared with external guests. The default is Zimbra.

  • Free/Busy Interop to used to set up free/busy scheduling across a mix of ZCS servers and third party email servers in the domain. See Making Free/Busy View Available

  • Zimlets.  Zimlets that are available on the domain are listed. You can manage which Zimlets are enabled and are mandatory.

  • Use the Themes tab to quickly change the colors used for the client interface. See Quick Custom Color

  • Account Limits on the domain. You can set the maximum number of accounts that can be provisioned on the domain and maximum accounts per COS.
